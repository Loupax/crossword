# German Crossword Puzzle System

A self-contained toolchain for generating, solving, and publishing German crossword puzzles (Schwedenrätsel). Supports both manual creation in the browser and fully automated daily generation via Claude.

## Requirements

- **Node.js** v18+ (for the CLI generator and deploy script)
- **Python 3.10+** (for `generate-daily` and `generate_words`)
- **Anthropic API key** (for `generate-daily` and `generate_words`)

Install the Python dependency:

```bash
pip install anthropic
```

No `npm install` needed — the CLI uses only Node.js built-ins.

---

## Setup

1. Clone the repo and `cd` into it.
2. Create a `.env` file with your Anthropic key:

```
ANTHROPIC_API_KEY=sk-ant-...
```

3. Source it before running any Python scripts:

```bash
source .env
# or, for a single command:
set -a && source .env && set +a
```

Make sure the shebang scripts are executable (they should already be):

```bash
chmod +x generate-daily generate_words gen serve scripts/deploy-daily
```

## Quick Start

### Open the interactive creator

```bash
./serve
# then open http://localhost:8080 in your browser
```

### Generate a puzzle from a CSV file

```bash
echo 'HAUS,"Wo man schläft"
HUND,"Bester Freund des Menschen"
KATZE,"Miaut"' | node cli/index.js --size 7 --strategy greedy
```

This writes `cw_<hex>.html` and `cw_<hex>.txt` to the current directory.

### Generate a daily puzzle automatically

```bash
set -a && source .env && set +a
./generate-daily
```

This calls Claude to generate fresh German word/hint pairs, builds a 15×15 puzzle, and saves it to `puzzles/backlog/`.

### Publish the next puzzle

```bash
./scripts/deploy-daily
```

This takes the first puzzle from `puzzles/backlog/`, copies it to `docs/index.html`, and commits + pushes to GitHub Pages.

## Tools Overview

### Web Creator (`index.html`)

An interactive browser-based crossword builder. Open via `./serve` or directly in a browser.

**Features:**
- Add words and hints manually
- Import words by pasting CSV into the import textarea
- Click grid cells to place words (horizontal or vertical)
- Auto-place all words automatically
- Resize the grid (N×N)
- Export as **JSON** (raw puzzle data) or **standalone HTML** (playable puzzle, no server needed)
- State is auto-saved to `localStorage` — your work survives page reloads

**Workflow:**
1. Type a word and its hint, click "Add"
2. Click a cell in the grid, then click a word in the list to place it — or use "Auto-place all"
3. Click "Export HTML" to generate a shareable self-contained puzzle file

---

### Web Player (`player.html`)

A puzzle solver. Load a puzzle exported from the creator or generated by the CLI.

**Loading a puzzle:**
- Drag-and-drop or upload a `cw_*.html` file
- Or paste raw JSON into the textarea

**Controls:**
- Click a cell to focus it; type to enter a letter
- Arrow keys navigate between cells
- Click a clue to jump to and highlight that word
- Answers are validated in real time (green = correct, red = wrong) without revealing them — they are stored as SHA-256 hashes
- "Show answer" reveals a single word; "Reveal all" fills the entire grid
- Progress is saved automatically to `localStorage` and restored on reload

---

### CLI Generator (`cli/index.js`)

Reads word/hint pairs from **stdin** (CSV format), places them on a grid, and outputs a playable HTML file.

```bash
node cli/index.js [options] < words.csv
```

**Options:**

| Flag | Default | Description |
|------|---------|-------------|
| `--strategy` | `backtrack` | Placement engine: `backtrack`, `greedy`, `simple`, `template` |
| `--size` | `15` | Grid size (N×N) |
| `--output-dir` | `.` | Directory for output files |
| `--exclude` | `used_words.txt` | File of already-used words to skip; placed words are appended here automatically |
| `--template-file` | *(none)* | ASCII template file (required for `--strategy template`) |
| `--timeout` | `5m` | Time budget for the solver. Formats: `30s`, `2m`, `5m`, or milliseconds |
| `--help` | | Print usage |

**Outputs:**
- `cw_<hex>.html` — fully self-contained playable puzzle
- `cw_<hex>.txt` — list of placed words (one per line)

**Examples:**

```bash
# Quick 7×7 puzzle using the greedy engine
node cli/index.js --size 7 --strategy greedy < words.csv

# Large 15×15 puzzle with a 10-minute solver budget
node cli/index.js --size 15 --strategy backtrack --timeout 10m < words.csv

# Output to a specific directory
node cli/index.js --output-dir puzzles/backlog < words.csv

# Using a custom template layout
node cli/index.js --strategy template --template-file my.template < words.csv

# Disable exclusion tracking entirely
node cli/index.js --exclude /dev/null < words.csv
```

### Daily Puzzle Generator (`generate-daily`)

Generates a fresh German crossword puzzle automatically using Claude, tracking which words have been used to avoid repetition.

```bash
./generate-daily [options]
```

**Requires:** `ANTHROPIC_API_KEY` in the environment.

**Options:**

| Flag | Default | Description |
|------|---------|-------------|
| `--level` | `B1` | CEFR level for the answer words |
| `--hint-level` | `B2` | CEFR level for the clue language |
| `--theme` | `None` | Vocabulary theme (e.g. `Travel`, `Food and Cooking`) |
| `--output-dir` | `puzzles/backlog` | Where to write the puzzle HTML |
| `--history-file` | `word_history.csv` | CSV file tracking used words and dates |
| `--size` | `15` | Grid size N×N |
| `--strategy` | `backtrack` | Placement engine |
| `--timeout` | `5m` | Solver time budget |

**What it does:**

1. Reads `word_history.csv` and builds an exclusion set of previously used words
2. Calls Claude Opus 4.6 to generate 30 fresh word/hint pairs (10 short, 10 medium, 10 long), passing the exclusion list so Claude avoids those words
3. Filters the returned words a second time against the exclusion set (safety net)
4. Pipes the filtered CSV into `node cli/index.js`
5. Reads the placed-word list from the CLI's `.txt` output
6. Appends each placed word with its hint and today's date to `word_history.csv`

**Examples:**

```bash
# Default: B1 words, B2 hints, no theme, 15×15 grid
./generate-daily

# A1 vocabulary, A2 hints, food theme
./generate-daily --level A1 --hint-level A2 --theme "Food and Cooking"

# Smaller puzzle for testing
./generate-daily --size 7 --timeout 1m

# Custom output directory and history file
./generate-daily --output-dir /tmp/puzzles --history-file /tmp/history.csv

# B2 words with a travel theme, faster greedy strategy
./generate-daily --level B2 --hint-level B2 --theme Travel --strategy greedy --timeout 2m
```

**Word history format (`word_history.csv`):**

```
HAUS,"Wo man schläft",2026-03-21
HUND,"Bester Freund des Menschen",2026-03-21
```

> Note: A TTL purge script (to remove old entries after N days) has not been implemented yet. To manually clear the history, delete or truncate `word_history.csv`.

### Bulk Word Generator (`generate_words` / `gen`)

Generates large CSV vocabulary files across multiple themes and CEFR levels in batch. Useful for building a word pool to use with the CLI.

```bash
./generate_words --hint-level B1 --batch-size 100 --themes "Travel" "Food and Cooking"
```

**Options:**

| Flag | Description |
|------|-------------|
| `--hint-level` | CEFR level for hints (required) |
| `--batch-size` | Words to generate per theme per CEFR level (default: 10) |
| `--themes` | One or more theme names (space-separated; quote multi-word themes) |

**Output:** One CSV file per theme/level combination:

```
crossword_Travel_A1_B1.csv
crossword_Travel_A2_B1.csv
crossword_Travel_B1_B1.csv
```

**The `gen` wrapper** runs `generate_words` across all 31 built-in themes at once:

```bash
set -a && source .env && set +a
./gen
```

Built-in themes:
`Travel`, `Food and Cooking`, `Daily Routine`, `Nature and Weather`, `Emotions`, `Home and Furniture`, `Household Chores`, `Clothing and Fashion`, `Shopping and Money`, `Family and Relationships`, `Professions and Work`, `School and Education`, `Describing People`, `Hobbies and Free Time`, `Sports and Fitness`, `Health and Human Body`, `Holidays and Celebrations`, `City and Infrastructure`, `Public Transport`, `Animals and Pets`, `Time and Calendar`, `Technology and Communication`, `Arts and Culture`, `Media and Entertainment`, `Politics and Society`, `Banking and Post Office`, `Crime and Justice`, `Geography and Countries`, `Environment and Sustainability`, `Science and Space`, `Tools and DIY`

**Then pipe a generated CSV into the CLI:**

```bash
node cli/index.js --size 15 < crossword_Travel_B1_B1.csv
```

### Deploy Script (`scripts/deploy-daily`)

Rotates the next puzzle from the backlog to the live GitHub Pages site.

```bash
./scripts/deploy-daily
```

**What it does:**

1. Reads `puzzles/backlog/` and picks the first `.html` file (FIFO — alphabetical order)
2. Copies it to `docs/index.html` (served by GitHub Pages)
3. Moves the file to `puzzles/completed/`
4. Runs `git add docs/index.html`, commits with message `chore: daily puzzle rotation`, and pushes

Exits with an error if the backlog is empty.

**Typical daily workflow:**

```bash
# 1. Generate a puzzle and add it to the backlog
./generate-daily

# 2. When ready to publish, deploy the next puzzle
./scripts/deploy-daily
```

## CSV Word Format

All scripts consume and produce CSV in this format:

```
WORD,"Hint text"
```

Rules:
- Words must be **uppercase A–Z only** — no umlauts, no spaces
- Expand umlauts before use: `Ä→AE`, `Ö→OE`, `Ü→UE`, `ß→SS`
  - Example: `FRÜHLING` → `FRUEHLING`
- Hints should be in **double quotes**
- No header row

Example file:

```csv
HAUS,"Wo man schläft"
HUND,"Bester Freund des Menschen"
SONNE,"Scheint am Tag"
FRUEHLING,"Jahreszeit nach Winter"
SCHULE,"Wo Kinder lernen"
```

## Word History & Deduplication

`word_history.csv` is a plain CSV file that records every word placed by `generate-daily`:

```
WORD,"HINT",YYYY-MM-DD
```

On each run, `generate-daily` reads this file to build an exclusion list that is passed to Claude. Claude avoids generating those words, and the script also filters out any that appear anyway.

To reset the history (allow words to be reused):

```bash
# Clear completely
> word_history.csv

# Or remove entries older than 90 days manually
```

> A TTL purge script is planned but not yet implemented.

## Placement Strategies

The CLI supports four placement engines selectable via `--strategy`.

### `backtrack` (default)

Recursive backtracking with a time budget. Tries all candidate positions for each word, tracking the best partial solution found. Returns the best result before the timeout expires.

- Best coverage
- Slowest; use `--timeout` to control duration
- Good for large grids or high-quality output

```bash
node cli/index.js --strategy backtrack --timeout 10m < words.csv
```

### `greedy`

Scores every valid position for each word and picks the best one greedily (no backtracking). Scoring favours:
1. Maximum intersections with already-placed words
2. Future connectivity (how many remaining words could use the new letters)
3. Placement close to grid centre

- Fast, good coverage
- Deterministic within a run

```bash
node cli/index.js --strategy greedy < words.csv
```

### `simple`

Places the first word at the grid centre, then iterates remaining words in order and places each at the first valid position with at least one intersection.

- Fastest
- Lowest coverage
- Useful for quick previews

```bash
node cli/index.js --strategy simple < words.csv
```

### `template`

Places words into a pre-defined ASCII art grid layout. See [Template Strategy](#template-strategy) below.

---

## Template Strategy

Define a fixed grid layout with an ASCII file:

- `.` = white cell (letter can go here)
- `#` = black cell (blocked)

Example (`5x5.template`):

```
.....
.###.
.....
.###.
.....
```

The engine parses all horizontal and vertical runs of white cells (length ≥ 2) as slots, then assigns words by length using backtracking.

```bash
node cli/index.js --strategy template --template-file 5x5.template < words.csv
```

A default template is included at `cli/strategies/default.template`.

## Deployment to GitHub Pages

The project deploys to GitHub Pages from the `docs/` directory.

1. Enable GitHub Pages in your repo settings → set source to `docs/` on `main`
2. Fill the backlog:
   ```bash
   ./generate-daily   # repeat as many times as needed
   ```
3. Deploy one puzzle at a time:
   ```bash
   ./scripts/deploy-daily
   ```

The live puzzle is always at `docs/index.html`. Completed puzzles are archived in `puzzles/completed/`.

```
crossword/
├── index.html              # Interactive web creator
├── player.html             # Web puzzle player
├── generate-daily          # Daily puzzle generation script (Python)
├── generate_words          # Bulk word/hint CSV generation (Python)
├── gen                     # Wrapper: runs generate_words across all 31 themes

├── word_history.csv        # Tracks used words to prevent repetition
│
├── cli/
│   ├── index.js            # CLI entry point (reads CSV stdin, outputs HTML)
│   ├── html-exporter.js    # Generates self-contained playable HTML
│   ├── candidate-selector.js  # Word scoring and bucketing (anchors/mediums/glue)
│   ├── state-manager.js    # Loads/saves the used-words exclusion file
│   └── strategies/
│       ├── backtrack.js    # Recursive backtracking engine
│       ├── greedy.js       # Greedy scoring engine
│       ├── simple.js       # Simple first-valid-position engine
│       ├── template.js     # Fixed-layout template engine
│       └── default.template  # Default ASCII grid template
│
├── scripts/
│   └── deploy-daily        # Rotates backlog → docs/index.html → git push
│
├── puzzles/
│   ├── backlog/            # Generated puzzles waiting to be deployed
│   └── completed/          # Deployed puzzles archive
│
└── docs/
    └── index.html          # Live GitHub Pages puzzle (updated by deploy-daily)
```
